# Counter Scam Agency — 遊戲企劃草案

## 1. 遊戲定位
- **題材**：防範詐騙的角色扮演（RPG）
- **玩家身分**：新晉「反詐情報官」，帶著 AI 夥伴在城市中協助市民識破詐術。
- **體驗目標**：
  1. 在案件推理中學習防詐知識。
  2. 享受劇情、角色養成與工具解鎖的成就感。
  3. 建立可長期擴充的案件/圖鑑系統。

## 2. 核心循環
1. 接收案件或市民請求。
2. 調查：蒐集對話、帳務、社群、設備等線索。
3. 判讀：辨識詐術、估計危險程度。
4. 處置：指導 NPC、阻止匯款、釣魚回擊或聯繫警方。
5. 回饋：結算聲望、技能點、情報素材，更新「詐術圖鑑」。
6. 成長：升級技能樹、解鎖新工具與高難度事件。

## 3. 系統支柱
| 系統 | 說明 | 範例 |
| --- | --- | --- |
| 案件調查 | 呈現 NPC 案例，玩家以工具檢視資料並推理 | 假客服、投資詐騙、AI 語音詐術 |
| 技能養成 | 四大技能樹：社交洞察、金融偵測、數據取證、心理韌性 | 解鎖語音比對、交易異常分析、心理疏導 |
| 情報擴散 | 破案後撰寫警示公報或開直播分享，提升城市安全度 | 城市安全度越高，詐騙成功率越低 |
| 圖鑑收集 | 每種詐術會記錄徵兆、危害與防範步驟 | 可供玩家/AI 夥伴在後續案件中引用 |

## 4. 章節示例
1. **新手訓練**：假冒客服／OTP 詐騙，教授基本辨識。
2. **市區巡防**：網拍詐騙、投資群組，需要多線索交叉。
3. **情感攻防**：愛情詐騙、親友冒名，著重情緒與語音分析。
4. **高科技對決**：深偽影音、AI 語音，加入迷你分析遊戲。
5. **跨國詐團**：建立關係圖譜與多 NPC 協調任務。

## 5. AI 夥伴與工具
- AI 夥伴可訓練不同模組（語音、文字、社群爬蟲、金融模型）。
- 工具例：關鍵字辨識、反詐熱線、釣魚網址掃描、聲紋比對。
- 工具有冷卻或資源限制，要求玩家策略運用。

## 6. 指標與難度
- **聲望**：代表玩家可信度，影響 NPC 是否第一時間求助。
- **城市安全度**：越高則詐騙事件成功率下降，但詐團會升級手法。
- **壓力值**：處理大量案件會累積壓力，需要休息/活動降低。

## 7. MVP 建議
1. 兩種案件：假客服、投資社群。
2. 基礎調查介面：訊息檢視 + 可疑標注 + 單一技能樹。
3. 成功/失敗結算：聲望、圖鑑、下一案件解鎖。
4. CLI 或簡易 UI 原型，先驗證推理流程與回饋循環。
